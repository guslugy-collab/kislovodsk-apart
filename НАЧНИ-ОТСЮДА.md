## 3. Жёсткие правила
- **Цифры только честные** (доход 8–12%, аренда 5–15 тыс ₽/сутки). Это защита клиента от ФАС/претензий. Не завышать.
- **Тёмный премиум-стиль** (`--bg:#05101E` + золото `#C9A96E`, Playfair+Inter) — НЕ менять на светлый.
- **Демо-плейсхолдеры** (название ВЕРШИНА, фото из интернета, цены, адрес, контакты, `STOCK`, документы-образцы, `TOUR.youtube`) — заменяются реальными данными объекта ПЕРЕД боевым запуском.
- **Без живых существ** в иконках; иконки — тонкие SVG, не эмодзи.

## 4. Решения владельца (не передоговаривать)

## 5. Стек и запуск

## 6. Стиль и ритуал

## 7. ПОДВОДНЫЕ КАМНИ (без них ИИ ломает работу)
- ⚠️ **ДВЕ СЕССИИ пишут в один `index.html`.** Вторая — Dispatch с телефона, коммитит под git-именем «Albert». Обе в ОДНОЙ локальной папке → чужие несохранённые правки лежат в общем рабочем дереве.
  - Перед правкой: `git fetch` + заново прочитать `index.html`.
  - Коммитить **мелко и сразу пушить**; перед коммитом смотреть `git diff --stat`.
  - Если своя фича изолирована — по возможности `git add <конкретные пути>`, не `git add -A` (иначе захватишь чужую незакоммиченную работу).
  - Чужую историю **не переписывать** force-push'ем. Идеально — стройка из ОДНОГО окна за раз.
- ⚠️ **Скриншотер встроенного браузера виснет.** Проверять через `javascript_tool` + `read_console_messages` (computed-стили, наличие элементов). Реальные скрины — только Chrome headless (`--headless=old`).
- ⚠️ Если владелец прислал скрин с «выцветшим светлым фоном» — это глюк тёмного/ночного режима его браузера (Brave force-dark / расширение), а НЕ сайт. Сайт единообразно тёмный.
