## 7. ПОДВОДНЫЕ КАМНИ (без них ИИ ломает работу)
- ⚠️ **ДВЕ СЕССИИ пишут в один `index.html`.** Вторая — Dispatch с телефона, коммитит под git-именем «Albert». Обе в ОДНОЙ локальной папке → чужие несохранённые правки лежат в общем рабочем дереве.
  - Перед правкой: `git fetch origin` + заново прочитать `index.html`. Ветка одна — `main`.
  - Коммитить **мелко и сразу пушить**; перед коммитом смотреть `git diff --stat`.
  - Если своя фича изолирована — по возможности `git add <конкретные пути>`, не `git add -A` (иначе захватишь чужую незакоммиченную работу в свой коммит).
  - При расхождении с origin: `git pull --rebase origin main`. Чужую историю **не переписывать** force-push'ем.
  - Как понять, активен ли телефон: `git log -1 --format="%cr %an %s"` — если недавний коммит от «Albert», сессия работает; тогда бери **изолированную** задачу (новый блок/новые файлы), а не hero/меню/глобальное. Идеал — стройка из ОДНОГО окна за раз.
- ⚠️ **Скриншотер встроенного браузера виснет.** Проверять через `javascript_tool` + `read_console_messages` (computed-стили, наличие элементов). Реальные скрины — только Chrome headless (`--headless=old`).
- ⚠️ **Кириллические имена файлов:** `Write` создаёт и `bash cat/cp` читают их корректно; инструмент Read по прямому кириллическому пути может не найти файл — тогда читай через `bash cat` или редактируй через `Write`.
- ⚠️ Если владелец прислал скрин с «выцветшим светлым фоном» — это глюк тёмного/ночного режима его браузера (Brave force-dark / расширение), а НЕ сайт. Сайт единообразно тёмный.

## 9. КАРТА index.html (чтобы не искать вслепую)
Один файл. Порядок: `<style>` в `<head>` → разметка секций в `<body>` → `<script>` в конце.
- **Секции `<body>` сверху вниз:** preloader, header/nav + мобильное меню, `#hero` (слайд-шоу), `#benefits`, `#about`, `#advantages`, `#tour` (видео-облёт), `#catalog` (фильтр + счётчик `#stock`), `#gallery`, отделка/меблировка, `#investor` (график + B2B + халяль), `#calc`, `#location` (переключатель Схема/Карта), `#docs`, `#faq`, `#contact` (форма), footer. Плюс оверлеи: `#quizOv`, `#docOv`, `#vidOv`, мобильная панель `#mbar`.
- **Данные/логика в `<script>`:** объект переводов `I18N` (`ru`/`en`; правки текста — тут, по ключам `data-i18n`), данные `UNITS` (каталог), `STOCK` (счётчик дефицита), `QUIZ` (шаги квиза), `DOCS` (документы), `TOUR` (`youtube:''` → вставить ID ролика), `FAQ`, `PLAN` (метки локации). Инициализация — в блоке `DOMContentLoaded`.
- **Тексты меняются в двух местах:** статичный текст в HTML (fallback) И ключ в `I18N.ru`/`I18N.en` — правь оба, иначе при переключении языка вернётся старое.
